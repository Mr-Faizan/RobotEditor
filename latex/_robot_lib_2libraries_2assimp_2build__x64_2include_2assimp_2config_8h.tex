Specifies the blob name, assimp uses for exporting. 

Some formats require auxiliary files to be written, that need to be linked back into the original file. For example, OBJ files export materials to a separate MTL file and use the {\ttfamily mtllib} keyword to reference this file.

When exporting blobs using \#\+Export\+To\+Blob, assimp does not know the name of the blob file and thus outputs {\ttfamily mtllib \$blobfile.\+mtl}, which might not be desired, since the MTL file might be called differently.

This property can be used to give the exporter a hint on how to use the magic {\ttfamily \$blobfile} keyword. If the exporter detects the keyword and is provided with a name for the blob, it instead uses this name. \Hypertarget{_robot_lib_2libraries_2assimp_2build__x64_2include_2assimp_2config_8h_a6d57233a2400383f433998a15633d7ff}\index{config.h@{config.h}!AI\_CONFIG\_EXPORT\_FBX\_TRANSPARENCY\_FACTOR\_REFER\_TO\_OPACITY@{AI\_CONFIG\_EXPORT\_FBX\_TRANSPARENCY\_FACTOR\_REFER\_TO\_OPACITY}}
